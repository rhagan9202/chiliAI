# Documentation Archive

This directory contains historical documentation snapshots, completed plans, and superseded planning material. These files are preserved for traceability, but they are not the source of truth for current implementation status.

Use active docs first:

- `docs/architecture.md` for system design and module boundaries.
- `backend/README.md` and `chili_app/README.md` for current setup and surface area.
- `docs/todos_and_stubs_audit_2026-05-05.md` for the current TODO/stub inventory.

Archived planning files under `docs/archive/planning/` may contain stale acceptance criteria, old issue references, or completed story prompts.
